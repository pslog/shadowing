Bạn là senior full-stack engineer. Hãy giúp tôi xây dựng một web app tên tạm là **Shadow IT Japanese**.

## 1. Mục tiêu sản phẩm

Đây là web app luyện shadowing tiếng Nhật chuyên ngành IT dành cho người Việt làm IT hoặc học IT tiếng Nhật.

Trọng tâm của app:

* Luyện phát âm tiếng Nhật qua từng câu thoại.
* Chủ đề bài học xoay quanh môi trường IT: daily standup, code review, bug report, API, database, deployment, BrSE, interview, keigo công ty.
* Tạo thói quen học mỗi ngày bằng daily mission, streak, XP, level.
* User nói từng câu, được chấm điểm, câu nào đạt từ 80 điểm trở lên thì pass.
* Pass đủ daily target thì giữ streak.
* Pass hết câu trong bài thì hoàn thành bài.

App không cần làm quá phức tạp ngay. Ưu tiên làm bản chạy được với flow học hoàn chỉnh.

### 3.5 Lesson Player

Route: `/lessons/[id]`

Đây là màn hình quan trọng nhất.

Hiển thị:

* Lesson title
* Progress: ví dụ “Câu 7/24”
* Current sentence tiếng Nhật
* Vietnamese translation nếu có
* Audio player
* Nút nghe câu gốc
* Nút nghe chậm 0.75x nếu dễ làm
* Nút record
* Nút retry
* Điểm lần gần nhất
* Trạng thái pass/fail của câu
* Danh sách câu bên cạnh hoặc phía dưới để thấy câu nào pass/chưa pass

Flow:

1. User chọn câu.
2. User nghe audio gốc.
3. User bấm record.
4. App ghi âm bằng MediaRecorder.
5. User dừng record.
6. Upload recording lên Supabase Storage.
7. Gọi scoring function.
8. Hiển thị điểm:

   * Pronunciation
   * Speed
   * Intonation
   * Total
9. Nếu total >= 80 thì câu pass.
10. Nếu total < 80 thì hiển thị feedback ngắn và yêu cầu luyện lại.
11. Khi pass đủ 5 câu trong ngày thì daily mission completed.
12. Khi pass hết câu trong bài thì lesson completed.

Ví dụ kết quả pass:

“83 ✅ Pass”

Breakdown:

* Phát âm: 85
* Tốc độ: 79
* Ngữ điệu: 82

Ví dụ fail:

“74 🟡 Almost”

Feedback:

* “Bạn chỉ còn thiếu 6 điểm để pass.”
* “Vấn đề chính là tốc độ hơi nhanh.”
* “Hãy nghe lại câu gốc rồi thử nói chậm hơn một chút.”

### 3.6 Progress Page

## 5. Scoring logic

Làm scoring đơn giản trước, nhưng code cần tách module để sau này thay bằng AI/API thật.

File gợi ý:

* `lib/scoring/pronunciation.ts`
* `lib/scoring/speed.ts`
* `lib/scoring/intonation.ts`
* `lib/scoring/total.ts`

### Công thức total

Total Score = Pronunciation 50% + Speed 30% + Intonation 20%

```ts
totalScore = Math.round(
  pronunciationScore * 0.5 +
  speedScore * 0.3 +
  intonationScore * 0.2
)
```

### Pronunciation score

Bản đầu nếu chưa có speech-to-text thật thì làm mock hoặc input transcript giả.

Yêu cầu:

* Tạo function nhận `targetText` và `spokenText`.
* Tính similarity đơn giản bằng Levenshtein distance hoặc token similarity.
* Return score 0–100.

Nếu chưa có spokenText thì trả random realistic score 60–95 để test flow.

### Speed score

Input:

* originalDurationSeconds
* userDurationSeconds

Rule:

* Nếu ratio từ 0.85 đến 1.15: 90–100
* Nếu ratio từ 0.70 đến 1.30: 70–89
* Ngoài khoảng đó: 40–69

ratio = userDurationSeconds / originalDurationSeconds

Nếu không có originalDurationSeconds thì dùng mock score 70–95.

### Intonation score

Bản đầu mock score 65–95.

Sau này có thể thay bằng audio energy contour/pitch contour.

### Feedback

Tạo function generateFeedback(scores) trả về feedback ngắn.

Rule:

* Nếu total >= 90: “Rất tốt, câu này nghe tự nhiên.”
* Nếu total >= 80: “Pass rồi. Có thể luyện thêm để nói tự nhiên hơn.”
* Nếu pronunciation thấp nhất: “Vấn đề chính là phát âm. Hãy nghe lại câu gốc và nói rõ từng cụm.”
* Nếu speed thấp nhất: “Vấn đề chính là tốc độ. Bạn đang nói quá nhanh hoặc quá chậm so với câu mẫu.”
* Nếu intonation thấp nhất: “Vấn đề chính là ngữ điệu. Hãy chú ý nhịp ngắt và độ lên xuống của câu.”

## 6. Pass/fail logic

Một sentence pass khi:

```ts
totalScore >= passScore
```

Mặc định passScore = 80.

Khi sentence pass:

* Tạo sentence_attempts.
* Cập nhật lesson_progress.
* Cập nhật daily_missions.
* Cộng XP.
* Nếu pass hết sentence trong lesson thì set lesson_progress.status = completed.
* Nếu daily mission đạt 5 câu thì set is_completed = true và cập nhật streak.

## 7. Streak logic

Daily mission chính:

* User chỉ cần pass 5 câu bất kỳ trong ngày để giữ streak.
* Không bắt buộc hoàn thành cả bài.
* Hoàn thành cả bài chỉ là bonus.

Khi daily_mission từ chưa completed chuyển sang completed:

* Kiểm tra profiles.last_completed_date.

Logic:

1. Nếu last_completed_date là hôm qua:

   * current_streak += 1
2. Nếu last_completed_date là hôm nay:

   * không tăng nữa
3. Nếu last_completed_date null:

   * current_streak = 1
4. Nếu last_completed_date cũ hơn hôm qua:

   * current_streak = 1

Sau đó:

* last_completed_date = hôm nay
* longest_streak = max(longest_streak, current_streak)

Quan trọng:

* Một ngày chỉ được tăng streak 1 lần.
* Pass thêm nhiều câu trong cùng ngày không tăng streak nữa.
* Pass thêm thì vẫn cộng XP.

## 8. XP và level logic

XP rule:

* Pass 1 câu: +5 XP
* Pass câu total >= 90: +8 XP thay vì +5
* Hoàn thành 1 bài: +50 XP
* Hoàn thành daily mission: +100 XP
* Streak đạt mốc 7 ngày: +300 XP bonus

Level formula:

```ts
level = Math.floor(totalXp / 500) + 1
```

Level title:

* Lv. 1: IT Shadow Beginner
* Lv. 5: Junior Dev Speaker
* Lv. 10: Daily Meeting Survivor
* Lv. 15: Code Review Speaker
* Lv. 20: BrSE Candidate
* Lv. 30: Japanese Tech Communicator
* Lv. 50: Tech Lead Speaker

## 9. UX quan trọng

App phải tạo cảm giác user muốn quay lại.

Nguyên tắc UI:

* Dashboard phải đơn giản, tập trung vào streak và mission hôm nay.
* Daily target phải nhỏ: pass 5 câu.
* Sau khi user pass 5 câu, hiện message: “Bạn đã giữ streak hôm nay 🔥”
* Sau mỗi attempt, show điểm và 1 feedback chính, không show quá nhiều lỗi.
* Nếu user cải thiện điểm, show: “+9 điểm so với lần trước 🎉”
* Nếu fail, không làm user nản. Dùng text nhẹ:

  * “Gần đạt rồi”
  * “Bạn chỉ còn thiếu 6 điểm”
  * “Thử lại một lần nữa nhé”
* Nếu mất streak, không trách user. Message:

  * “Không sao, hôm nay pass 5 câu để bắt đầu lại.”
